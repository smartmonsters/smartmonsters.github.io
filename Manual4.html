With coin specifications very similar to Huntercoin (see huntercoin.org), the game is based on a heavily modified Huntercoin version 1.2 ("less hearts" fork), and can be played using only the coin wallet (Qt-Client). There are no "servers" other than the cryptocurrency nodes. All player moves are valid transactions, stored in the blockchain.

POW mining for testnet

As in Huntercoin, about 80 percent of all coins are mined by the game, 20 percent through POW mining.

Testnet POW mining can be done from within the Qt-client: in Debug Window, use "setgenerate true" to start mining, and "setgenerate false" to stop. To switch from default algo sha256 to scrypt, put "algo=scrypt" in smartmonsters.conf. POW mining is needed to generate blocks, the game can't do this, even if it generates most of the coins.

Note that default smartmonsters.conf ("algo=scrypt" and "gen=1") starts the scrypt POW miner. It will mostly idle though, generating coins slowly.


Changelog (v2.2.03)


  • Starting with version 2.2.03, when new versions contain any change (or bugfix) that constitutes an hardfork, this change will be dormant until activated by the voting system.
  • Players saved by their Amulet or Ring of Life Saving go into vacation mode instead of being teleported to their color's base. (Rationale: make it easier to just skip playing for some time without penalty)
  • Players on vacation are ignored when determining player count and player combat score for balancing against monsters.
  • Pathfinding for player characters will always check both ways of their color's 2-way teleporter and use the shortest (not necessarily least dangerous) path to and from battlefield areas.


Wishlist

Trustless, decentralised trading

A special state ("Trader") for player characters in which they:

  • Can't leave safe zones and thus can't die.
  • Can send/receive looted coins to/from other players.

And another state ("Broker") for player characters in which they additionally:

  • Must maintain a minimum amount of looted coins as collateral. The voting system can take these coins away.

Now decentralised trading of coins can be made trustless without something like the complicated 2-of-3 multigateway that was once used to trade Nxt against BTC. A simple (1-of-1) gateway would be sufficient because whoever operates it is on the hook with all collateralized coins.

Trustless trading of levelled characters between players can be automated within the client.


CPU optimisation

Characters who are on the run due to failed morale check must only try to change retreat target every block if their tactical situation would get worse otherwise.

No need to count votes every block.


Better map

Increase number of coin spawning areas to accommodate more players. Split large areas that have coins more than 10-12 tiles away from their flag marker.

Map connectivity: dead ends are a bad thing, even large ones (several areas). Need "beta competition" for real coins to actually test this.


Code base

Client forked from newest Huntercore instead of an old version of Huntercoin.

New gamestate with reserve fields, for the game itself, for each player, each character, and (like "destruct") the moves, in preparation for changes during beta test.
