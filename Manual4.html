Windows Qt-Client

SmartMonsters-beta-2.2.05-20190122.zip

In order to use the shortcuts, extract the folder "smartmonsters" to C:\ so that the executable is at C:\smartmonsters\smartmonsters-qt.exe

If started normally, it will connect to the online beta testnet.


Testnet-in-a-box

A TiaB is included in the Windows Qt-Client.

To create a local testnet-in-a-box, use one of the "offline - node1 ..." shortcuts and one of the "offline - node2 ...". The "generate" versions start with activated CPU miner.


Source

https://github.com/smartmonsters


POW mining for testnet

About 80 percent of all coins are mined by the game, 20 percent through POW mining.

Testnet POW mining can be done from within the Qt-client: in Debug Window, use "setgenerate true" to start mining, and "setgenerate false" to stop. To switch from default algo sha256 to scrypt, put "algo=scrypt" in smartmonsters.conf. POW mining is needed to generate blocks, the game can't do this, even if it generates most of the coins.


Changelog (v2.2.03alpha)


  • Starting with version 2.2.03, when new versions contain any change (or bugfix) that constitutes an hardfork, this change will be dormant until activated by the voting system.
  • Players saved by their Amulet or Ring of Life Saving go into vacation mode instead of being teleported to their color's base. (Rationale: make it easier to just skip playing for some time without penalty)
  • Players on vacation are ignored when determining player count and player combat score for balancing against monsters.
  • Pathfinding for player characters will always check both ways of their color's 2-way teleporter and use the shortest (not necessarily least dangerous) path to and from battlefield areas.


Changelog (v2.2.04alpha)


  • Resurrected monsters start in their color's base instead of a semi-random harvest area.
  • Four "Starting Zones" for low level characters have been created near the map borders, equally distant to color bases. Everyone can enter, but higher level chars will be temporarily reduced to level 3.
  • Arbalests have always a strength of 2, and a range of 4 tiles if wielded by level 3 characters.
  • Fix: if characters are fleeing, their AI will no longer randomly decide to go after coins.


Changelog (v2.2.05beta)

Beta testnet is a different chain than alpha testnet.

  • Fixed algorithm to automatically generate the Obstacle Map from visual data.
  • Merchant NPCs no longer stand in the way of typical player paths.
  • Fixed remaining unreachable coins.
  • South exit of the center "marketplace" is now open.
  • Locked coin amount when creating a new player character now depends on block height, going down every 10000 blocks from 100 coins at block 0-10000 to 20 coins at block 50000.


Wishlist

Map size is the bottleneck limiting the number of players. It must become larger without increasing the memory footprint.

Map connectivity, and also weapon and item effects are still mostly "random" and need balancing through open playtesting.

The game need more items without making the learning curve for players even steeper.

Game client's blockchain and network related functions must migrate from current code base to that of the newest bitcoin version.

Current artwork is mostly placeholders, and the name "Smartmonsters" is perhaps not perfect...
